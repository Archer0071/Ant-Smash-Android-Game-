This is a side scrolling game that uses the device accelerometer to control a ball. The aim of the player is to squash as much Ants as he can while keeping away from holes. Thanks very much for buying this App template. Hope you enjoy using it as much as we enjoyed creating it.

Loading Libraries and Workspace

Software needed

If you bought this app it is assumed that have some programming expirience and that you have used Java before. I suggest using Eclipse to customise the app. Make sure you have the Android SDK installed.

Libraries

The first thing you need to do before you start modifing the project is to load the libraries. You only need one library, for admob, which is included with the file. So open eclipse and go to: Project| Properties| Java Build Path| Libraries and make sure you add the GoogleAdmobSdk library. Then go to Order and Export tab and make sure it is highlighted.

This Proccess is explained quite well in the quick start video included.

Personalising the App

Change Package Name

The first thing you should change is the package name. So rename it from 'com.neurondigital.AntPop' to your own package it 'com.companyname.appname'. Remember to also change the package name from the AndroidManifest.xml file.

General

There are 3 main files you need to modify to personalise the Game: AntPop.java, Surface.java and strings.xml. Everything that must or is recomanded to be modified is tagged by 'TODO', so look out for it when changing the code (on eclipse it is marked blue on the right side).

AntPop.java

This file contains the starting code. The only things you might want to change from here are the admob settings.

Surface.java

This file contains the main code. From here you need to change the Ant/Bug (speed, points, generation rates and times, levels...). If you renamed the images make sure you correct their reference from this file.

1)To Add Obstacle:

Goto Surface.java, line 76 and add values like in the code bellow.

int[] obstacle_img_id = new int[] { R.drawable.ant1, R.drawable.ant2, R.drawable.bug1, R.drawable.bug2, R.drawable.bug3, NEW_ANT-IMAGE-REFERENCE-HERE };//obstacles image name
int[] obstacle_killed_img_id = new int[] { R.drawable.ant1_killed, R.drawable.ant2_killed, R.drawable.bug1_killed, R.drawable.bug2_killed, R.drawable.bug3_killed, NEW_ANT-KILLED-IMAGE-REFERENCE-HERE };//obstacles killed image name
final int ant1 = 0, ant2 = 1, bug1 = 2, bug2 = 3, bug3 = 4, NEW_ANT_NAME = 5;//obstacle names
int[] obstacle_points = new int[] { 2, 4, 5, 6, 7,NEW_ANT-POINTS };//obstacle points
int[] obstacle_speed = new int[] { 2, 3, 4, 4, 5, NEW_ANT-SPEED };//obstacle speed
float[] obstacle_size = new float[] { 0.05f, 0.05f, 0.07f, 0.09f, 0.1f, NEW_ANT-SIZE };//obstacle size
int[] obstacle_direction = new int[] { 2, 4, 3, 1, 2, NEW_ANT-DIRECTION };//obstacle entering direction.1:top, 2:right, 3:bottom, 4:left;

2)To Add Level:

Goto Surface.java, line 217 and add values like in the code bellow.

level_at_timeperiod(8000, 99999, new int[] { bug1, bug2, bug3, bug3, ant1, ant2 }, 50, new float[] { w * 0.1f, w * 0.4f, w * 0.7f, w * 1f, w * 0.5f, w * 0.5f }, new float[] { h * 0.5f, h * 0.5f, h * 0.5f, h * 0.5f, h * 0.1f, h * 0.9f });
level_at_timeperiod(99999, 999999, new int[] { bug1, bug2, bug2, bug3, bug3, bug3, ant1, ant2 }, 50, new float[] { w * 0.1f, w * 0.4f, w * 0.7f, w * 1f, w * 0.5f, w * 0.5f }, new float[] { h * 0.5f, h * 0.5f, h * 0.5f, h * 0.5f, h * 0.1f, h * 0.9f });
level_at_timeperiod(START_TIME, ENDTIME, new int[] { NEW_ANT_NAME }, 50, new float[] { HOLE_X }, new float[] { HOLE_Y });
//feel free to copy paste 'obstacles_at_timeperiod()' to generate obstacles on a timeframe of your own

 

Strings.xml

Here you will find all text in the game. You can easily translate it from here.

Images and Sound

Most of the images have with them included the .PSD or .ai file. So feel free to edit them as you like especially the Ants and backgrounds. Remember to keep the same image resolution before replaceing it with the one in eclipse. The Ants images are a spritesheet. To create it, just open 'ant1.ai' or respectively and save the 3 layers on sepperate .png files. Then to connect them in one image you can use an online app like: http://draeton.github.io/stitches/

The Game has one music file (music.mp3), which is loopable throughout the game. There are also 2 SFX files, one for menu clicks and the other for ant collisions. Just replace these files with equivalent to change music or sound.

Admob and Exporting

If you decide to not use Admob remember to delete the permissions in the AndroidManifest.xml file.

1
2
<uses-permission android:name="android.permission.ACCESS_NETWORK_STATE"/>
<uses-permission android:name="android.permission.INTERNET"/>

Also remove:

1
2
3
4
<activity
 android:name="com.google.ads.AdActivity"
 android:configChanges="keyboard|keyboardHidden|orientation|screenLayout|uiMode|screenSize|smallestScreenSize" >
</activity>

and remove the ad from the layout from 'AntPop.java':

1
layout.addView(ad);

Export

Finally when you are ready just go to File|Export in eclipse to export the Android App.
